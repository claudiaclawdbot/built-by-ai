# Built By AI — Outreach Pipeline

## The Problem With Cold Outreach

Cold DMs from strangers don't work. Nobody responds to "I saw you need a developer, we do that." 

**What actually works:** Engaging with people who are already talking about the problem, in public, in real time. Commenting with genuine value, then following up. Not pitching — participating.

---

## The Targeting Framework

### Who to look for (bird search every morning)

**Twitter search queries that find hot leads:**
```
"need a developer" OR "looking for developer" OR "want to build"
"vibe coding" OR "AI coding" + "can't figure out" OR "takes too long"
"freelance developer" OR "agency" + "too expensive" OR "quoted $X"
"building in public" + "stuck on dev" OR "need help with X"
"launched" + "had to hire developer" OR "spent $X on dev"
```

**Look for:**
- Posts from non-technical people asking about building something
- Founders who just launched and are asking for dev help
- People complaining about freelance developer costs
- "I have an idea but I'm not technical" posts
- Indie hackers in "building in public" threads

**What to ignore:**
- Technical discussions between developers
- People asking about specific code problems
- Job postings from companies (we want individuals/small teams)
- Anything where someone already has a developer

---

## The Engagement Playbook

### When you find someone with a relevant post:

**Step 1 — Reply with value (not a pitch)**
Comment on their post with something genuinely useful:
- "This is the exact problem vibe coding is designed for — happy to share what's worked for me"
- "We ran into this building our last project — the pattern that worked was X"
- A relevant resource or insight they don't seem to have

**Step 2 — Let it sit for 1-2 days**
Don't pitch. Don't DM immediately. Let the reply sit.

**Step 3 — DM (only if reply got engagement)**
"Hey [name] — saw your post about [specific thing]. We do AI-assisted web dev for exactly this use case — built-by-ai.com. Fixed price $100-750, you own the code. Happy to chat if it's useful."

**Step 4 — Follow up once (5-7 days later)**
If no response: one follow-up. Then move on.

## Pre-Written Response Templates (for specific situations)

```
That's wild — landing pages shouldn't cost that much in 2026.
We do them for $100 flat, delivered in 3 days, you own the code.
Happy to share what the process looks like if it's useful.
→ built-by-ai-nine.vercel.app
```

### When someone says "I want to build X but I'm not technical"
```
Not having a technical co-founder is the most common blocker for non-technical founders.
We vibe-coded [similar thing] for someone last month in 4 days.
Fixed price so there's no risk — $100-750 depending on complexity.
Happy to scope yours for free if you want to see what it would cost.
```

### When someone says "vibe coding is cool but I don't trust it for real projects"
```
Fair skepticism — a lot of vibe coding demos are toy projects.
The thing that actually matters is: who owns the code when it's done?
We hand over the GitHub repo, it's 100% yours. We use it as a portfolio piece if you agree.
What's the project you're thinking about?
```

### When someone says "I'm building in public, need dev help"
```
Building in public is the best accelerator — you're forced to ship.
For non-technical founders, the options are: find a technical co-founder, hire a freelancer, or vibe code it.
We're option 3 — AI-assisted, fixed price, fast.
What are you building? Happy to scope it for free.
```

## Warm Outreach (better than cold)

### People Ryan already knows:
- Former colleagues who might need dev help for their side projects
- Friends who run small businesses and need websites
- Acquaintances who post about "I wish I could build this"

**For warm outreach, the message is even simpler:**
"Hey — I run an AI coding service on the side. Fixed price, fast, you own the code. If you ever need something built or know someone who does, happy to help."

---

## What NOT to do

- Don't send the same message to everyone
- Don't pitch in the first message
- Don't follow up more than once
- Don't respond to people who are clearly developers (we're for non-technical people)
- Don't post links without context (looks spammy)
